# 家庭大富翁 — 详细需求功能文档（完整版）

> 3D Multiplayer Monopoly
> 本版本在原始文档基础上，整合了「第一人称沉浸式街道漫游」功能（第十三章），并修订了 9.6 摄像机模式。

## 一、游戏规则引擎

### 1.1 回合流程

```
lobby → rolling → buying/stock/wheel/quiz/debt → awaitEnd → rolling → ...
                                                                    ↓
                                                                 ended
```

| 阶段 | 说明 |
|------|------|
| `lobby` | 等待玩家加入，选择主题/难度 |
| `rolling` | 当前玩家可掷骰子、建房/卖房、乘高铁、进出内城、监狱操作 |
| `buying` | 走到无主地产，需决定购买或放弃 |
| `stock` | 走到股市格（24号），可买卖股票后点"完成" |
| `wheel` | 走到大转盘（39号），转动转盘 |
| `quiz` | 回合开始时12%概率触发知识问答 |
| `debt` | 现金不足支付，需卖房或宣布破产 |
| `awaitEnd` | 移动完成后，可建房/卖房后结束回合 |
| `ended` | 产生赢家，可重新开始 |

### 1.2 骰子与移动

- 两个六面骰（1-6），掷出双数可再掷一次（地面环）
- **连续三次双数**：直接进监狱（12号位）
- 地面环（0-47）按最短路径移动；内城环（48-71）循环移动
- 经过 GO（0号）领取工资；切换到内城后从对应入口开始

### 1.3 地产与租金

| 类型 | 租金规则 |
|------|----------|
| **普通地产** | `rent[房屋数]`，0栋时若拥有整组颜色则双倍 |
| **铁路** | 按持有数量：`[25, 50, 100, 200]`（上海版额外×1.5倍组合） |
| **公用事业** | 1个=4×骰点，2个=10×骰点 |
| 所有租金 × 难度租金倍率 |

**10组颜色地产**：brown / lightblue / teal / pink / orange / red / yellow / plum / green / blue

### 1.4 建房规则

- 最高 5 栋（5=酒店），需拥有整组颜色
- **均匀建造**：组内任意地块房屋数不能超过最低的1栋以上
- **均匀出售**：从房屋最多的地块开始卖，不能造成1栋以上的差距
- 卖出退还房价的一半（向下取整）

### 1.5 监狱

- 12号位=监狱，36号位=进监狱
- 罚金 $50，最多关3回合
- 选项：付$50 / 用出狱卡 / 掷双数（失败3次强制付$50）
- 出狱卡可累积

### 1.6 债务与破产

- 现金不足时进入 `debt` 阶段，可卖房筹资
- 宣布破产：所有地产+现金转给债主（无债主则释放）
- 仅剩1名未破产玩家时立即获胜
- 达到回合上限时，净资产最高者获胜

### 1.7 回合维护费

新回合开始时扣除 `现金 × drainPct`：
- 简单 0% · 标准 3% · 困难 5% · 极限 8%

### 1.8 净资产计算

`现金 + 地产原价总和 + 房屋半价总和 + 股票市值总和`

## 三、机会与公益金卡牌

- **机会**：基础14张 + 主题额外
- **公益金**：基础16张 + 主题额外
- 效果包括：移动到某格、获得/支付现金、进出监狱、维修费、向每位玩家收/付钱、免费出狱卡、最近铁路/公共事业

## 四、股票市场

- **18只股票**：FOOD/RETAIL/AUTO/REIT/ENERGY/MINING/AGRI/BANK/INSURE/TECH/AI/CHIP/NET/PHARMA/BIO/GOLD/LUXURY/AIR
- **交易费**：3%，最低 $5
- **价格波动**：每回合随机浮动（±漂移值）
- **分红**：每回合概率选中一只被持有的股票，按分红率派息
- **随时交易**：非 lobby/ended 阶段可随时打开交易面板
- **成本核算**：按比例计算成本基础，追踪已实现盈亏和分红收益

- 木质边框（60×60）+ 金饰

### 9.5 房屋与酒店
- 绿色房屋 / 红色酒店，Box+屋顶锥体

## 十、程序化城市

### 10.0 素材来源策略

程序化生成负责**布局与数量**（建筑位置、密度、随机变化），具体的**建筑/道具模型主体**优先复用外部 CC0（公共领域）素材，而非完全手工建模，以控制美术成本并保证风格统一：

| 用途 | 来源 | 格式要求 |
|------|------|----------|
| 商业建筑（10.4节店面） | Kenney City Kit (Commercial) | GLB，导入前压缩 |
| 住宅建筑 | Kenney City Kit (Suburban) | GLB，导入前压缩 |
| 模块化楼层拼接 | Kenney Modular Buildings / Quaternius LowPoly Buildings | GLB |
| 道路/路灯/护栏/施工道具 | Kenney City Kit (Roads)，或整理好的 GLB 转换包 | GLB，可直接加载 |
| 街道模块套件 | Quaternius Modular Streets Pack | GLB |
| 表面材质（沥青/砖墙/混凝土的 diffuse/normal/roughness/AO） | Poly Haven（100% CC0） | 贴图直接替换 `materials.js` 中的程序化噪声贴图 |

- 所有外部模型统一转换/导出为 **GLB** 格式，与 Three.js/`GLTFLoader` 原生兼容
- 导入前必须经过**减面 + 压缩**（Draco 或 Meshopt 编码），否则第一人称漫游模式下帧率会明显下降，具体流程见技术方案文档
- 风格约束：建筑/道具素材统一选用低多边形（low-poly）美术基调，避免混用不同精度/风格来源导致场景观感不一致
- 上海/东京主题的地标建筑（10.5节，如东方明珠、浅草寺等）因无法在公共素材库中找到匹配模型，仍需手工建模或委托定制，不在此素材复用范围内

## 十三、第一人称街道漫游

### 13.1 功能定位

- 与回合制游戏状态机（第一章）**完全解耦**：漫游是纯客户端本地表现层功能，棋子的逻辑位置仍由服务端权威决定
- 触发方式：玩家在 HUD 视角切换按钮（12.2节）新增"🚶 漫游"选项，随时可从俯瞰/跟随视角切入，不影响回合流程
- 单人视角体验：玩家在自己设备上自由控制第一人称角色于棋盘街道间行走，其位置**不广播**给其他玩家（不占用联机同步带宽，不影响游戏逻辑）
- 退出漫游：按 ESC 或点击"返回棋盘"按钮，相机平滑过渡回原视角模式

### 13.2 街道场景布局

- 街道两侧对称布置建筑地块，交替生成**商店**与**住宅**两类立面，比例约 6:4（商店略多，符合城市街道观感）
- 单条街道横截面（由内向外）：
  ```
  建筑退线 → 人行道(宽2单位) → 路缘石 → 车行道(宽4单位，双向) → 路缘石 → 人行道 → 建筑退线
  ```
- 建筑间距采用不规则抖动（propJitter 工具），避免网格化排列的"程序生成感"
- 沿用10.2节的10种建筑风格分层，漫游场景中额外要求：
  - 商店类建筑必须在临街首层渲染10.4节描述的店面装饰（遮阳篷、招牌、户外桌椅等）
  - 住宅类建筑首层渲染门廊/台阶/花箱，不出现商业招牌

### 13.3 商店与住宅的区分规则

| 类型 | 首层特征 | 建筑高度范围 | 占比 |
|------|----------|--------------|------|
| 商店 | 大玻璃橱窗+招牌+遮阳篷+户外座位（可选） | 2-6层 | 60% |
| 住宅 | 门廊/台阶+窗户+花箱/阳台 | 2-8层 | 40% |

- 生成时按种子（seed）确定性分配类型，保证同一场景多次进入布局一致
- 商店子类型（见10.4节）随机分配：咖啡馆/餐厅/商店/医院/面包店/书店/办公室

### 13.4 第一人称控制

| 输入 | 桌面端 | 移动端 |
|------|--------|--------|
| 视角旋转 | 鼠标拖拽/指针锁定 | 触屏拖拽 |
| 点击移动 | 点击地面瞬移至目标点 | 点击地面瞬移至目标点 |
| 退出漫游 | ESC / 按钮 | 按钮 |

- 移动方式：点击地面瞬移，无需WASD自由行走
- 眼睛高度：约1.6-1.7单位，随地面高度（人行道/车行道存在高差时）自动吸附
- 碰撞：瞬移目标点需在可行走区域内，避免穿模

### 13.5 与已有系统的联动

- 昼夜循环（9.7节）与天气系统（9.8节）在漫游模式下保持运行，光照/雾效实时影响街道视觉
- 夜间发光系统（10.11节）在漫游视角下的招牌、路灯、车灯发光效果应更突出（近距离观察）
- 环境音效（11.2节）音量随漫游深入街道内部产生简单的空间衰减/混响暗示（可选增强项）

### 13.6 舒适性与可访问性

- 默认视场角（FOV）75°，可在设置中调节 65°-90°
- 点击地面瞬移作为唯一移动方式，降低晕动症风险
- 瞬移过渡可加简短淡入淡出，避免视觉跳跃感

### 13.7 性能要求

对应9.9节画质模式表格新增的漫游相关行（建筑渲染半径 / 行人车辆密度 / 碰撞体精度）。
